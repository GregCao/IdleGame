# 放置游戏 - 游戏设计文档 (GDD)

## 1. 游戏概述

### 游戏名称
**放置江湖** (待定)

### 游戏类型
放置类挂机RPG

### 核心玩法
玩家角色自动战斗，收集金币，升级属性，挑战更高波次。

### 目标用户
- 休闲玩家
- 上班族（碎片时间）
- 无长期在线条件的用户

### 平台目标
- 微信小程序（首发）
- 抖音小游戏（海外）
- 海外Web/Steam（后续）

## 2. 核心循环

```
┌─────────────────────────────────────────┐
│                                         │
│   ┌─────┐    攻击     ┌──────┐         │
│   │玩家 │ ──────────→ │ 怪物 │         │
│   └─────┘    击杀     └──────┘         │
│      ↑                       │         │
│      │ 升级                  │ 掉落金币 │
│      │                       ↓         │
│   ┌─────┐               ┌──────┐       │
│   │等级 │ ←──────────── │ 金币 │       │
│   └─────┘    消耗升级   └──────┘       │
│                                         │
└─────────────────────────────────────────┘
```

### 离线循环
- 关闭游戏 → 计算离线时间
- 重新打开 → 领取离线收益
- 激励视频 → 双倍领取

## 3. 数值设计

### 玩家属性
| 属性 | 初始值 | 每级成长 | 公式 |
|------|--------|---------|------|
| 攻击力 | 10 | +5 | base + (level-1) × 5 |
| 生命值 | 100 | +20 | base + (level-1) × 20 |
| 暴击率 | 10% | - | 固定 |
| 暴击伤害 | 150% | - | 固定 |

### 升级消耗
```
升级金币 = level × 100
例: Lv1→Lv2 = 200金, Lv10→Lv11 = 1000金
```

### 怪物属性
```
怪物生命 = playerLevel × 50 × 波次系数
怪物攻击 = playerLevel × 2 × 波次系数
金币奖励 = playerLevel × 20 × 波次系数

波次系数 = 1 + (wave × 0.3)
```

### 离线收益
```
离线金币 = 攻击力 × 0.5 × 离线秒数
离线上限 = 8小时
```

## 4. 战斗系统

### 战斗节奏
- 玩家攻击间隔: 1秒
- 怪物攻击间隔: 2秒
- 击杀后延迟: 0.5秒生成新怪物

### 波次机制
- 无限波次
- 波次越高，怪物属性越强
- 波次奖励加成: 1 + (wave × 0.05)

## 5. 变现设计

### 激励视频场景
| 场景 | 奖励 | 说明 |
|------|------|------|
| 离线双倍 | 离线收益×2 | 核心留存点 |
| 加速 | 跳过冷却 | 可选 |
| 额外金币 | 5分钟金币产出 | 日常活跃 |

### IAP商品
| 商品 | 价格 | 类型 | 效果 |
|------|------|------|------|
| 新手礼包 | $0.99 | 非消耗 | 5000金 + 10倍伤害(永久) |
| 月卡 | $4.99 | 订阅 | 每日1000金 |
| 金币小礼包 | $0.99 | 消耗 | 10000金 |
| 金币中礼包 | $4.99 | 消耗 | 50000金 |
| 金币大礼包 | $9.99 | 消耗 | 200000金 |
| 去除广告 | $1.99 | 非消耗 | 永久去广告 |

### 定价策略
```
免费玩家 → IAA维持
月卡玩家 → IAP稳定收入
大R玩家 → 抽卡+加速
```

## 7. 技术架构

### 引擎
- Unity 2022 LTS
- 目标平台: 微信小程序

### 脚本结构
```
GameManager (总控)
├── PlayerManager (玩家数据)
├── BattleManager (战斗逻辑)
├── EconomyManager (金币经济)
├── AdManager (广告控制)
├── IAPManager (支付)
├── UIManager (界面更新)
└── SaveManager (存档)
```

### 数据持久化
- PlayerPrefs (轻量存档)
- 存档键: PlayerData
